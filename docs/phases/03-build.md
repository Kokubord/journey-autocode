# Build — implementar o plano

Build **executa** o que a Discovery entregou. Não é momento de reinventar o âmbito inteiro (mudanças reais passam por scope-review).

## Disciplina útil

- Uma **sub-fase** = uma fatia mergeável (ex. uma user story), não a fase inteira numa branch gigante  
- CI real assim que a stack estiver definida (placeholder de Foundation não conta como “CI verde”)  
- Cada decisão material → ADR  

## Emergente

Trabalho que nasce no meio do Build deve ser formalizado (feature/tasks no roadmap) **antes** de virar código solto.
