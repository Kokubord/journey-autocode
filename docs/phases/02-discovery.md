# Discovery — produto, escopo e decisões

Aqui o produto se define: visão consolidada, Constituição (completa), ADRs (incluindo stack), specs + plans + tasks (Spec Kit), e confirmação de escopo.

## Documento de Visão

O rascunho do Warmup é **consolidado** em `docs/JOURNEY-VISION.md` (versão canónica no repo). O comando `/journey-discover` ajuda a conduzir workshops e a materializar decisões.

## Gate importante

A Discovery só está completa quando há plano completo das features priorizadas (spec + plan + tasks com ≥1 task cada) + constituição ratificada — antes de abrir o Build a sério. O Journey avisa se tentares avançar cedo demais.
