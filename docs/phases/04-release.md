# Release — preparar e publicar uma versão

Release trata de versionar, notas de lançamento, e o ritual de tornar uma versão utilizável (tag, branch `release/…`, checklist).

No AutoCode open-source, “publicar” pode significar:

- tag Git + notas;
- pacotes/wheels;
- ou, no ecossistema Journey, também a publicação do núcleo open (como este próprio repositório).

Usa `/journey-release-start` quando a fase estiver activa no teu projecto.
