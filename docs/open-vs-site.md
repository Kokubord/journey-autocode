# Open-source (AutoCode) vs Site Journey

Journey é **open-core**:

- **No teu repositório (este projecto Apache):** metodologia, templates, coletores, comandos `/journey-*`. Grátis e teu para sempre.
- **No Site (serviço):** consolidação do roadmap + Cockpit (vista rica). **1º projecto ligado = grátis**; projectos adicionais = plano pago (detalhe comercial fora deste repo).

## Podes viver só no AutoCode?

**Sim.** HANDOVER, ADRs, specs, agente e rituais funcionam sem Site. Perdes a vista hospedada e o sync automático do roadmap na nuvem — não perdes a metodologia.

## Quando vale a pena o Site?

Quando quiseres acompanhar o projecto numa UI (sponsor, equipa) sem ler só markdown, ou ligar vários artefactos num Cockpit.

## Fronteira técnica (resumo)

Os **dados crus** (HANDOVER, ADRs, `tasks.md`, métricas agregadas) ficam no **teu** git. O Site **lê** esses dados e monta a vista — não substitui a fonte no repo.
